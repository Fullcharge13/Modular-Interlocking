# 컨텍스트 예산 규칙 (Context Budget Rule)

> **결구 원칙**: 이 규칙은 불변입니다. 어떤 에이전트도 재해석하거나 우회할 수 없습니다.

---

## 핵심 원칙: 주두(柱頭) — 하중을 사방으로 분산

단일 LLM 호출에 모든 컨텍스트를 집중하는 것은 기둥을 계속 깎는 것과 같다.
기둥이 부러지기 전에 주두를 올려야 한다.

---

## 컨텍스트 예산 임계값

| 임계값 | 상태 | 행동 |
|---|---|---|
| 0–40% | 🟢 정상 | 자유롭게 작업 |
| 40–50% | 🟡 주의 | 불필요한 파일 로드 중단 |
| 50–65% | 🟠 경고 | `compact-trigger` skill 실행 |
| 65–80% | 🔴 위험 | 현재 작업 체크포인트 저장 후 압축 |
| 80%+ | ⛔ 중단 | 즉시 압축, 압축 전 새 작업 시작 금지 |

---

## 서브에이전트 컨텍스트 규칙

### 전달 허용 항목
```
✅ 현재 태스크의 Spec (목표, 인터페이스 정의)
✅ 관련 파일 최대 3개
✅ 실패 메시지 (있을 경우)
✅ 이전 시도의 요약 (원문 아님)
```

### 전달 금지 항목
```
❌ 전체 대화 히스토리
❌ 무관한 파일 내용
❌ 이미 완료된 태스크의 세부 내용
❌ 중복 컨텍스트 (같은 내용 반복)
```

---

## 컨텍스트 압축 트리거 조건

다음 중 하나라도 해당되면 즉시 압축을 고려한다:

1. **연구/탐색 완료 후** → 구현 시작 전
2. **마일스톤 완료 후** → 다음 마일스톤 시작 전
3. **디버깅 완료 후** → 기능 개발 재개 전
4. **실패한 접근 포기 후** → 새 접근 시작 전
5. **컨텍스트 50% 도달 시** → 자동 트리거

### 압축 금지 조건
- 구현 **도중** (변수명, 파일 경로, 부분 상태 손실 위험)
- 테스트가 RED 상태일 때

## 모델 선택 규칙

```python
# 컨텍스트 크기에 따른 모델 라우팅
if task_complexity == "simple" and context_pct < 30:
    model = "haiku"      # 비용 최소화
elif task_complexity == "standard":
    model = "sonnet"     # 기본값
elif task_complexity == "architectural":
    model = "opus"       # 아키텍처, 심층 추론만
```

---

## 위반 시 처리

이 규칙을 위반하는 패턴이 감지되면:
1. 현재 작업을 체크포인트에 저장
2. `agents/context-guardian`에 위임
3. 압축 후 재개
