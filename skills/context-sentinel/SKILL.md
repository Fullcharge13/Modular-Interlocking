# Context Sentinel (컨텍스트 감시병)

> **주두 원칙**: 이 skill은 기둥이 부러지기 전에 하중을 감지한다.
> 넓은 면적의 주두처럼, 컨텍스트 상태를 사방에서 관찰하여 집중 방지.

---

## 역할

세션 중 컨텍스트 사용량을 능동적으로 감시하고, 임계값 도달 전에 경고를 발행한다.
에이전트가 "이미 넘쳤을 때" 반응하는 것이 아니라, "넘치기 전에" 행동하게 한다.

---

## 발동 조건

다음 중 하나가 감지되면 자동 발동:

```
- 컨텍스트 사용량 40% 도달 시 (주의 단계)
- 대화 턴이 15회를 초과할 때
- 로드된 파일이 5개를 초과할 때
- 단일 응답이 2,000 토큰을 초과할 때
- 동일 컨텍스트가 3회 이상 반복될 때 (중복 감지)
```

---

## 실행 프로토콜

### Step 1: 상태 스냅샷

```
현재 컨텍스트 구성 파악:
□ 총 사용량 추정 (%)
□ 로드된 파일 목록
□ 활성 태스크 수
□ 중복 컨텍스트 존재 여부
□ 마지막 압축 이후 경과 턴 수
```

### Step 2: 하중 분류

```python
# 컨텍스트 구성요소별 비중 분류
HEAVY   = ["전체 파일 내용", "긴 에러 스택", "완료된 태스크 히스토리"]
MEDIUM  = ["현재 파일 관련 섹션", "요약된 이전 결과"]
LIGHT   = ["현재 태스크 Spec", "인터페이스 정의", "실패 메시지"]

# 목표: HEAVY 제거 → LIGHT + MEDIUM 유지
```

### Step 3: 경고 발행

```
🟡 주의 (40-50%):
   → "컨텍스트가 {X}% 사용 중입니다. 불필요한 파일 로드를 중단합니다."

🟠 경고 (50-65%):
   → "compact-trigger skill을 실행합니다."
   → compact-trigger로 위임

🔴 위험 (65-80%):
   → "현재 작업을 체크포인트에 저장합니다."
   → 체크포인트 저장 → 압축 → 재개

⛔ 중단 (80%+):
   → "새 작업 시작을 차단합니다. 즉시 압축이 필요합니다."
   → 모든 새 작업 차단 → 강제 압축
```

---

## 중복 컨텍스트 감지 패턴

```python
# 다음 패턴이 감지되면 중복으로 분류
DUPLICATE_SIGNALS = [
    "이미 앞서 언급했듯",
    "이전에 설명한 것처럼",
    "앞서 본 코드에서",
    # 같은 파일을 두 번 이상 전체 로드
    # 같은 에러 메시지가 3회 이상 등장
]

# 중복 감지 시: 원본 참조로 대체, 재전송 금지
```
